RECONSTITUTION UML — DIAGRAMMES RÉELS

Page exceptionnelle hors-mission. Compare visuellement les quatre propositions pour chaque question, choisis mentalement, puis dévoile la réponse en bas.

Légende UML : <|-- héritage (extends, triangle plein) · <|.. réalisation (implements, triangle pointillé) · *-- composition (◆ vie liée) · o-- agrégation (◇ référence) · --> association · ->> appel synchrone (séquence) · -->> retour (pointillé).

FRAGMENT 1 — Structure statique du système

Le Recopieur contient des Sortie (au moins une, plusieurs possibles, vie liée). Sortie est une interface. SortieAbstraite est une classe abstraite qui implémente Sortie et factorise l'attribut nom. SortieConsole et Antenne héritent de SortieAbstraite et fournissent publier(String). Un Capteur, distinct, implémente Sortie sans hériter de SortieAbstraite.
OPTION A
Composition pleine ◆ vers <<interface>> Sortie. SortieAbstraite réalise Sortie. SortieConsole et Antenne héritent de SortieAbstraite. Capteur réalise Sortie directement.
classDiagram
  direction TB
  class Recopieur
  class Sortie {
    <<interface>>
    +publier(String)
  }
  class SortieAbstraite {
    <<abstract>>
    #nom: String
  }
  class SortieConsole
  class Antenne
  class Capteur
  Recopieur "1" *-- "1..*" Sortie : contient
  Sortie <|.. SortieAbstraite
  SortieAbstraite <|-- SortieConsole
  SortieAbstraite <|-- Antenne
  Sortie <|.. Capteur
OPTION B
Agrégation creuse ◇ vers Sortie [0..*]. SortieConsole et Antenne implémentent Sortie directement (pas d'abstraite). Capteur extends Sortie.
classDiagram
  direction TB
  class Recopieur
  class Sortie {
    <<interface>>
    +publier(String)
  }
  class SortieConsole
  class Antenne
  class Capteur
  Recopieur "1" o-- "0..*" Sortie : référence
  Sortie <|.. SortieConsole
  Sortie <|.. Antenne
  Sortie <|-- Capteur
OPTION C
Association simple Recopieur ──> Sortie. SortieAbstraite extends Sortie (héritage de classe). SortieConsole, Antenne, Capteur héritent tous de SortieAbstraite.
classDiagram
  direction TB
  class Recopieur
  class Sortie
  class SortieAbstraite
  class SortieConsole
  class Antenne
  class Capteur
  Recopieur --> Sortie : utilise
  Sortie <|-- SortieAbstraite
  SortieAbstraite <|-- SortieConsole
  SortieAbstraite <|-- Antenne
  SortieAbstraite <|-- Capteur
OPTION D
Recopieur implements Sortie. SortieConsole et Antenne sont des attributs internes de Recopieur.
classDiagram
  direction TB
  class Recopieur {
    -console: SortieConsole
    -antenne: Antenne
  }
  class Sortie {
    <<interface>>
  }
  class SortieConsole
  class Antenne
  Sortie <|.. Recopieur
  Recopieur *-- SortieConsole
  Recopieur *-- Antenne
Bonne réponse : A. C'est la seule qui combine correctement (i) composition pleine ◆ [1..*] de Recopieur vers Sortie, (ii) réalisation pointillée SortieAbstraite ⤳ Sortie, (iii) héritage plein SortieConsole et Antenne de SortieAbstraite, (iv) Capteur réalise Sortie directement sans passer par l'abstraite — c'est précisément ce qui justifie d'avoir l'interface en plus de la classe abstraite.

Pourquoi pas B : agrégation creuse ◇ et [0..*] au lieu de [1..*], et pas de classe abstraite pour factoriser nom.
Pourquoi pas C : Sortie est traitée comme une classe et non comme une interface (héritage plein au lieu de réalisation). Et Capteur hérite de SortieAbstraite, ce que le scénario exclut.
Pourquoi pas D : Recopieur n'est PAS une Sortie : il contient des sorties. Inversion complète de la sémantique.

FRAGMENT 2 — Dynamique d'une publication (rendu PlantUML — conventions du prof N7)

Conventions PlantUML (à reproduire à la main sur la feuille) — c'est le rendu de référence de la correction officielle.
actor = silhouette en haut à gauche lifeline objet = rectangle crème nom:Classe flèche pleine = appel synchrone flèche pointillée = retour
«create» = instanciation (fait apparaître la lifeline en dessous) note jaune (post-it) = commentaire over X rectangle vertical = activation (la pile d'exécution sur cet objet) ✕ rouge en bas = destruction (destroy)
Un acteur Agent appelle recopieur.publier("alpha"). Le Recopieur itère sur ses Sortie. Pour la SortieConsole : publier("alpha") sans incident. Pour l'Antenne : publier("alpha") lève une PublicationException catchée localement et consignée. Le Recopieur retourne au final.
OPTION A
Agent appelle Recopieur.publier, qui appelle successivement SortieConsole.publier (retour normal) puis Antenne.publier (lève une exception catchée, lifeline maintenue). Recopieur retourne à Agent.
actor Agent
participant "r:Recopieur" as R
participant "sc:SortieConsole" as SC
participant "an:Antenne" as An
Agent -> R : publier("alpha")
activate R
R -> SC : publier("alpha")
activate SC
SC --> R
deactivate SC
R -> An : publier("alpha")
activate An
note over An : lève PublicationException
An --> R : catchée
deactivate An
note over R : consigne, continue
R --> Agent
deactivate R
OPTION B
Agent appelle Recopieur.publier. Le Recopieur lève une exception qui interrompt la séquence. Pas de retour à l'Agent.
actor Agent
participant "r:Recopieur" as R
Agent -> R : publier("alpha")
activate R
note over R : lève une exception
R -->x Agent : interrompu
deactivate R
OPTION C
Toutes les flèches sont inversées. Agent ← Recopieur, Recopieur ← SortieConsole, Recopieur ← Antenne.
actor Agent
participant "r:Recopieur" as R
participant "sc:SortieConsole" as SC
participant "an:Antenne" as An
R -> Agent : publier("alpha")
SC -> R : publier("alpha")
An -> R : publier("alpha")
OPTION D
Agent appelle SortieConsole directement. SortieConsole appelle Antenne. Antenne appelle Recopieur. Boucle de transmission en chaîne.
actor Agent
participant "sc:SortieConsole" as SC
participant "an:Antenne" as An
participant "r:Recopieur" as R
Agent -> SC : publier("alpha")
SC -> An : publier("alpha")
An -> R : publier("alpha")
Bonne réponse : A. C'est la seule fidèle au scénario : appel synchrone d'Agent vers Recopieur, qui délègue successivement à SortieConsole (retour normal pointillé) puis à Antenne (exception catchée localement, donc la lifeline du Recopieur n'est pas interrompue), puis retour final à Agent.

Pourquoi pas B : contredit le scénario (l'exception est catchée, donc Recopieur retourne bien à Agent).
Pourquoi pas C : sens des flèches inversé — en séquence, la flèche pleine va de l'appelant vers l'appelé, pas l'inverse.
Pourquoi pas D : les sorties ne se passent pas la main entre elles. C'est le Recopieur qui orchestre l'itération.
